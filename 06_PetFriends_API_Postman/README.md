# Тестирование API сервиса PetFriends в Postman

Учебный проект по тестированию REST API сервиса **PetFriends** с помощью Postman.  
Цель – отработать сценарное тестирование, работу с переменными коллекции и data-driven подход (CSV).

---

## Сценарий коллекции

Коллекция реализует цепочку из четырёх запросов:

1. **Получение API-ключа**  
   - Метод: `GET` / `POST` (в зависимости от учебного API)  
   - Входные данные: e-mail и пароль пользователя.  
   - Обязательный тест:
     - статус-код ответа `200`;
     - значение ключа сохраняется в переменную коллекции `auth_key`
       и используется в заголовках следующих запросов.

2. **Добавление питомца**  
   - Метод: `POST` `/api/pets` (или аналогичный эндпоинт).  
   - Тело запроса заполняется переменными (`name`, `animal_type`, `age` и др.).  
   - Обязательный тест:
     - статус-код ответа `200`;
     - `id` созданного питомца сохраняется в переменную `petid`;
     - имя питомца из ответа сохраняется в переменную `name`.

3. **Изменение имени питомца**  
   - Метод: `PUT` `/api/pets/{petid}`.  
   - В запросе используется переменная `petid`, новое имя передаётся через переменную.  
   - Обязательный тест:
     - статус-код ответа `200`;
     - проверка, что новое имя из ответа **не равно** старому имени;
     - новое имя сохраняется в отдельную переменную (для сравнения в последующих шагах).

4. **Получение списка своих питомцев**  
   - Метод: `GET` `/api/pets/my_pets`.  
   - Обязательный тест:
     - статус-код `200`;
     - значение переменной `name` коллекции равно полю `name` первого (`[0]`) питомца в ответе.

---

## Используемые инструменты

- **Postman (web/desktop)** – создание коллекции запросов и тестов;
- **Переменные окружения и коллекции** – хранение `auth_key`, `petid`, `name` и др.;
- **CSV-файл с тестовыми данными** – для параметризации POST/PUT запросов;
- **REST API PetFriends** – учебный backend от Skillfactory.

---

## Особенности реализации

- Все ключевые значения (ключ авторизации, id питомца, имена) передаются через переменные,  
  что позволяет переиспользовать коллекцию без ручного редактирования запросов.
- В CSV-файле предусмотрен кейс с **одинаковыми именами** для POST и PUT –  
  в этом случае тест для PUT-запроса ожидаемо падает (статус **FAILED**),  
  что демонстрирует работу негативного сценария.
- Коллекция может быть запущена в виде **сценарного теста** (Runner) с подгрузкой CSV.

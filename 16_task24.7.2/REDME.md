# Тестирование REST API: негативные и граничные тесты (Задание 24.7.2)

## В данном задании реализованы дополнительные методы библиотеки api.py, а также подготовлены 10 тест-кейсов для негативного, граничного и отказоустойчивого тестирования REST API сервиса.

- Описание задания

По условиям задания необходимо:

Изучить документацию REST API на сайте и определить методы, которые не реализованы в предоставленной библиотеке.

Добавить реализацию недостающих методов в файл api.py.

Разработать 10 тестов, проверяющих работу API в условиях некорректных, отсутствующих или граничных входных данных.

- Тесты должны проверять:

реакцию API на неверные параметры,

поведение при передаче слишком больших значений,

отсутствие обязательных параметров,

использование неверного ключа авторизации,

некорректные типы данных,

пустые поля,

попытку доступа без авторизации и т. д.

- Что включает работа

Реализованы отсутствующие методы API.

Написаны тесты для проверки устойчивости API к ошибкам пользователя.

Тесты оформлены в отдельном файле согласно структуре проекта.

Все тесты используют формат arrange — act — assert, обеспечивая читабельность и логическую структуру.

- Примеры типов протестированных сценариев

Неверный формат e-mail.

Передача строки вместо числа.

Передача отрицательных чисел.

Превышение ограничения длины полей.

Запрос без ключа авторизации.

Запрос с неверным API-ключом.

Отсутствие обязательного параметра.

Передача пустого тела запроса.

Использование неподдерживаемого HTTP-метода.

Попытка получить доступ к данным другого пользователя.

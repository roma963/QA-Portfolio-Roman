#  Проект тестирования сайта «ЭкскурсиУм»

Проект по функциональному тестированию фронт-сайта магазина экскурсий **https://excursium.com/**  
В рамках проекта выполнены как ручное тестирование, так и разработка автотестов на Python + Selenium.

---

##  Цель тестирования

Проверить работоспособность функций фронт-сайта и убедиться, что критические бизнес-процессы
(регистрация, авторизация, бронирование экскурсий, фильтрация, личный кабинет) работают корректно.

---

##  Задачи тестирования

- проверка соответствия реализованного функционала требованиям;
- обнаружение и идентификация дефектов;
- предотвращение критических дефектов в работе сайта;
- предоставление заинтересованным лицам достаточной информации о качестве продукта;
- снижение риска ненадлежащего качества работы сайта.

---

##  Объект тестирования

Фронт-сайт магазина экскурсий:

- лендинг;
- страница входа / регистрации;
- страница со списком экскурсий (фильтр);
- страница с конкретной экскурсией (онлайн-калькулятор, блок с ценой);
- страницы бронирования и оплаты;
- личный кабинет пользователя.

---

##  Типы и виды тестирования

Основной тип: **функциональное тестирование**

Проводились проверки:

- функциональных элементов на страницах;
- регистрации и авторизации пользователей;
- процесса бронирования и оплаты экскурсий;
- алгоритма фильтрации экскурсий;
- работы личного кабинета.

Методология:

- **Ручное тестирование** — проверки UI и пользовательского потока;
- **Автоматизированное тестирование** — автотесты для критических бизнес-процессов:
  регистрация, авторизация, бронирование, фильтрация.

---

##  Инструменты

- **Python 3**
- **Selenium WebDriver**
- **PyTest**
- **Chrome / Firefox**
- Таблицы (Google Sheets) — для чек-листов, тест-кейсов и баг-репортов

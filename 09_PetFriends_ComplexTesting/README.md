# Проект 9. Комплексное тестирование сайта PetFriends

## Описание проекта

Заказчик попросил провести комплексное тестирование сайта PetFriends:  
проверить пользовательский интерфейс, API, работу с данными и безопасность,  
а также зафиксировать все найденные дефекты в единой таблице.

Работа выполнялась по заданию модуля **15.4.1 «Мой первый заказ на тестирование сайта»**  
[Ссылка на "Требования к сайту"](https://docs.google.com/document/d/1otYGANpVn3AHh7YI41zhAdz_cnJW4Cvd/edit)

---

## Окружение

- ОС: **Windows 11**
- Браузеры: **Google Chrome**, **Mozilla Firefox**, **Microsoft Edge**
- Тестируемый ресурс: **PetFriends** (веб-версия)

---

## Используемые инструменты

- **Chrome DevTools** — проверка верстки, консоли, Network, Sources.
- **Postman / Swagger** — проверка API (GET / POST запросы).
- **Excel** — чек-лист, тест-кейсы и список багов.
- **Git / GitHub** — контроль версий, проверка истории коммитов и конфликтов.

---

## Цели проекта

- Оценить **функциональность** сайта и корректность работы ключевых сценариев.
- Проверить **кроссбраузерность** и **адаптивность** интерфейса.
- Провести базовую проверку **безопасности**:
  - использование HTTPS и SSL-сертификата;
  - хранение чувствительных данных (ключи, токены, логины/пароли);
  - поведение API при отсутствии CSRF-cookie и некорректных данных.
- Оценить **производительность** (скорость загрузки страниц и время ответа API).
- Проверить **UX / UI**: удобство фильтров, поиск, навигация, доступность (alt у картинок и т.п.).
- Проверить окружение разработки:
  - корректность истории коммитов;
  - наличие/отсутствие лишних файлов;
  - работу с ветками, конфликтами и чистой рабочей директорией.

## Основные найденные дефекты (кратко)

Полное описание см. в таблице (колонки *result* и *описание бага*).

1. **Отсутствие защищённого HTTPS-соединения (bug 4)**  
   Сайт открывается по HTTP и помечен как «не защищён», данные не шифруются.

2. **Невозможно полноценно тестировать API без CSRF-cookie (bug 5, bug 6)**  
   - GET/POST-запросы вне браузера возвращают 403, даже при корректных данных.  
   - Из-за этого невозможно проверить валидацию обязательных полей и обработку некорректных запросов (ожидаемый 400).

3. **Некорректная адаптивность интерфейса (bug 1)**  
   На мобильных устройствах изображения и элементы смещены, интерфейс не адаптируется под экран, пользоваться сайтом неудобно.

4. **Проблемы с формами и UX (bug 2, bug 3, bug 12, bug 15)**  
   - В фильтрах некорректный ввод не валидируется, ошибки не отображаются.  
   - Иногда отображается состояние «приют пуст», что вводит пользователя в заблуждение.  
   - Вкладка «Питомцы без дома» показывает не весь список животных.  
   - При поиске по слову «кот» карточка соответствующего питомца исчезает.

5. **Проблемы API и ответов сервера (bug 8, bug 9, bug 10)**  
   - При GET-запросе к `/api/pets` сервер возвращает HTML вместо JSON.  
   - Стрелка пагинации не подгружает список питомцев.  
   - Заголовок `Content-Type` и фактический формат ответа не согласованы.

6. **Недостатки по доступности и вёрстке (bug 11, bug 13)**  
   - У изображений питомцев отсутствует атрибут `alt`.  
   - Иконки соцсетей в футере не являются ссылками (`href` отсутствует), переход по ним невозможен.

7. **Производительность (bug 7)**  
   Главная страница загружается заметно дольше рекомендуемого времени (около 31 секунды при замере).

---

## Выводы

- Сайт содержит рабочий основной функционал, однако:
  - требуется внедрение **HTTPS + SSL** и доработка API-слоя;
  - необходимо улучшить **адаптивность** интерфейса и UX-сценарии (поиск, фильтрация, навигация);
  - есть проблемы с **производительностью**, которые влияют на комфорт работы пользователя;
  - часть сценариев тестирования безопасности и валидации на уровне API в текущем виде **заблокирована**.

Доработка перечисленных областей повысит безопасность сервиса, удобство использования и стабильность работы.

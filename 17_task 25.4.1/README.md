# Пользовательские сценарии и тестирование бизнес-логики API
(Практическое задание модуля 25.4.1)

## В этом проекте представлены пользовательские сценарии тестирования, направленные на проверку бизнес-логики REST API сервиса.
Ранее были реализованы атомарные тесты отдельных эндпоинтов, однако для полноты картины разработаны сценарии, которые эмулируют путь реального пользователя в системе.

- Цель работы

Проверить корректную работу сервисов не только по отдельности, но и в цепочке вызовов.

Убедиться, что пользовательские действия приводят систему в ожидаемое состояние.

Найти потенциальные ошибки, которые не проявляются при атомарном тестировании.

- Что реализовано

В проекте описаны пользовательские сценарии для API, включающие:

Проверку корректности авторизации и работы с токенами.

Создание сущностей и работу со связанными объектами (например: пользователь → питомец → фото питомца).

Проверку цепочки запросов, необходимых для выполнения полноценной пользовательской задачи.

Позитивные и негативные сценарии бизнес-логики.

Каждый тест-кейс оформлен в соответствии с требованиями модуля и включает:

-  Предусловия

-  Шаги выполнения

-  Ожидаемый результат

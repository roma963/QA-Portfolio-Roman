# Тестирование PetFriends: работа с ожиданиями Selenium
(Задание 30.5.1 — HW-04)

## В данном проекте реализованы автоматизированные UI-тесты для веб-приложения PetFriends, целью которых является проверка корректности отображения карточек питомцев и таблицы питомцев.
Особое внимание уделено использованию неявных (implicit) и явных (explicit) ожиданий Selenium WebDriver.

- Цель работы

Добавить неявные ожидания (implicit_wait) для проверки элементов карточек питомцев.

Добавить явные ожидания (WebDriverWait + expected_conditions) для проверки таблицы питомцев.

Повысить стабильность UI-тестов при разной скорости загрузки страницы.

- Что тестируется
Проверка карточек питомцев

В тест добавлены неявные ожидания всех элементов, включая:

1. Фото питомца

2. Имя

3. Порода

4. Возраст

## Используемые техники ожиданий
- Неявные ожидания (Implicit Wait)

Задаются один раз на весь драйвер.

Selenium ждёт появления элемента на странице в течение указанного времени.

Подход подходит для проверки статичных карточек питомцев.

- Явные ожидания (Explicit Wait)

Позволяют ожидать конкретное условие:

element_to_be_clickable

presence_of_element_located

visibility_of_all_elements_located

Используются для таблицы, где элементы загружаются динамически.

## Данный проект демонстрирует:

- умение корректно применять implicit и explicit wait;

- повышение стабильности тестов за счёт управления задержками;

- автоматизацию UI-проверок на уровне реального веб-приложения;

- соблюдение методологии написания стабильных автотестов.
